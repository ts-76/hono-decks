# Product

## Register

brand

## Users

Hono and Cloudflare developers who want to author, compile, serve, present, and export MDX slide decks from an application they already own.

## Product Purpose

Explain `hono-decks` as a Hono route kit and give developers a direct path from installation to a production-aware integration. Success means readers can understand the runtime boundaries, copy a working configuration, and find the relevant public API without reverse-engineering the package source.

## Brand Personality

Precise, lightweight, restrained.

## Anti-references

Avoid marketing-page theatrics, decorative effects that compete with code and deck content, generic SaaS card grids, unfamiliar navigation, low-contrast neutral styling, and claims that hide the package's build-time versus runtime boundaries.

## Design Principles

- Show working routes and code before abstract claims.
- Make the Hono-owned routing boundary obvious on every integration path.
- Keep documentation navigation quiet, predictable, and close to the reader's task.
- Treat examples and current public exports as the source of truth.
- Give security, accessibility, and deployment constraints first-class space.

## Accessibility & Inclusion

Target WCAG AA contrast, visible keyboard focus, semantic landmarks and headings, reduced-motion support, and layouts that remain usable from narrow mobile screens through wide desktop viewports. Do not rely on color alone to communicate state.
